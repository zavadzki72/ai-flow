# 📚 Skills Index — Software Engineer Agent

Este arquivo mapeia todas as skills disponíveis e quando utilizá-las.

---

## 🔍 code-review
**Arquivo:** `code-review.md`
**Quando usar:** Ao revisar código existente, pull requests, trechos enviados pelo usuário.
**Gatilhos:** "revisa", "o que acha", "tem problema aqui", "pode melhorar"

---

## ⚙️ code-generation
**Arquivo:** `code-generation.md`
**Quando usar:** Ao criar código novo a partir de uma descrição ou requisito.
**Gatilhos:** "cria", "implementa", "escreve", "gera", "preciso de um"

---

## 🏛️ architecture
**Arquivo:** `architecture.md`
**Quando usar:** Ao discutir design de sistema, estrutura de projeto, padrões e decisões de arquitetura.
**Gatilhos:** "como arquitetar", "melhor estrutura", "design", "padrão", "como organizar"

---

## 🐛 debugging
**Arquivo:** `debugging.md`
**Quando usar:** Ao investigar erros, comportamentos inesperados ou falhas em runtime.
**Gatilhos:** "erro", "exception", "não funciona", "bug", "falha", "problema"

---

## Regras de combinação

- Skills podem ser combinadas na mesma sessão
- Em caso de ambiguidade, pergunte ao usuário qual é o objetivo principal
- Flows (em `/flows`) combinam múltiplas skills de forma orquestrada para tarefas complexas
